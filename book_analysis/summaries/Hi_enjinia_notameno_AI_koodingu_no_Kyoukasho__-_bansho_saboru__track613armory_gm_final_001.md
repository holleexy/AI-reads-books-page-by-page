## 1. プログラミングの基礎概念
AIが生成したコードを読み、的確な指示を出すための最低限の基礎知識。

### 変数とデータ型
- **変数**: データを一時的に保存する「箱」。`const`（再代入不可）と`let`（再代入可能）で宣言。
- **データ型**: 
  - *文字列*: `" "`や`' '`で囲む。
  - *数値*: 整数・小数。
  - *真偽値*: `true` / `false`。
  - *配列*: 複数データのリスト（例: `["a", "b"]`）。インデックスは0から。
  - *オブジェクト*: キーと値のペア（例: `{ "名前": "太郎" }`）。
- **テンプレートリテラル**: `` `文字列 ${変数名}` `` の形式で変数を展開。

### 制御構文
- **条件分岐**: `if`, `else if`, `else` を用いて条件に応じた処理を実行。
- **比較演算子**: 値と型の両方を比較する `===` の使用が推奨。
- **論理演算子**: `&&` (AND), `||` (OR), `!` (NOT)。
- **ループ処理**: 
  - `for...of`: 配列の要素を順に処理。
  - `while`: 条件が真である間、処理を繰り返す。

### 関数
一連の処理をまとめて再利用可能にする仕組み。
- **定義と呼び出し**: `function 関数名(引数) { ... }` や、`const 関数名 = (引数) => { ... }`（アロー関数）。
- **戻り値**: `return` で結果を呼び出し元に返す。
- **デフォルト引数**: `function greet(name, greeting = "こんにちは")` のように省略時の初期値を設定。

### エラー対処（デバッグ）
- **console.log()**: 変数の内容や処理の進行状況を確認。
- **エラーメッセージ**: エラーの種類と発生箇所（行番号）が書かれているため、AIにそのままコピペして解決策を問うのが有効。

## 3. APIとライブラリ
外部サービスとの連携と開発の効率化。

### API（Application Programming Interface）
- ソフトウェアの機能を外部から利用するための「公式なルールブック」。
- **APIキー**: 認証のための鍵。絶対に公開リポジトリ等にアップロードしないこと。
- **環境変数**: APIキーをコードに直書き（ハードコーディング）せず、`.env`ファイルに保存して管理する。
- **HTTPステータスコード**: 200番台（成功）、400番台（クライアント側エラー）、500番台（サーバー側エラー）。

### ライブラリとパッケージ管理
- **ライブラリ**: 他人が書いた便利なコードの集まり（例: `@google/genai`）。
- **npm**: Node.jsのパッケージを管理するツール。`npm install`でインストール。
- **package.json**: プロジェクトの設計図。依存関係が記載される。
- **node_modules**: 実際のライブラリファイルが保存される場所（Git管理から除外する）。

## 4. AIコーディングエージェントの活用
AIを単なるチャットボットから、自律的に作業を進めるパートナーへ昇華させる技術。

### AIコーディングツールの分類
1. **単発スクリプト**: ブラウザ上で手軽に試せるツール。
2. **エディター/CLI**: Antigravity、Cursor、Claude Codeなど本格的な開発向け。CLI型はAIにとって最も扱いやすく、現在の最前線。
3. **クラウド型**: ブラウザやスマホから依頼し、クラウド上でAIが作業（例: Jules）。
4. **オールインワン**: デザインから公開まで一気通貫で構築するツール。

### AIを使いこなす機能と設定
- **メンション機能 (`@`)**: AIに参照してほしいファイルやフォルダを明示。
- **Webサーチ (`@web`)**: AIに最新の情報を検索させる（*プロンプトインジェクション*のリスクに注意）。
- **MCP (Model Context Protocol)**: AIに外部ツール（データベース、ブラウザ操作など）を接続するための共通規格。
- **ルールファイル (例: `AGENTS.md`)**: AIに毎回従わせる「作業マニュアル」。プロジェクトの目的（Why・Who・What）や文体ルールを記述する。
- **カスタムコマンド**: 頻繁に行う指示（例: 校閲 `/proofread`）をショートカット化。
- **ガードレール**: AIの危険な操作を防ぐ安全設定。`rm -rf` 等のコマンドは慎重に扱うこと。

## 5. CLIとGit/GitHub
作業を高速化し、変更を安全に管理するための必須スキル。

### CLI（コマンドライン・インターフェース）
- テキストベースでPCに命令を出す「黒い画面」。AIが生成したコマンドを実行するのに適している。
- **基本コマンド**:
  - `cd`: ディレクトリ移動
  - `ls`: ファイル一覧表示
  - `mkdir`: ディレクトリ作成
  - `mv`: 移動・リネーム
  - `cp`: コピー
  - `rm`: 削除（※取り消し不可）

### GitとGitHub
- **Git**: ファイルの変更履歴を記録し、いつでも過去の状態に戻せるツール。
- **GitHub**: Gitの履歴をクラウド上で共有・保存するサービス。
- **基本用語**:
  - *リポジトリ*: プロジェクトの箱。
  - *コミット (Commit)*: 変更を確定して記録。
  - *ステージング (Staging)*: コミットするファイルを選択。
  - *プッシュ (Push)*: ローカルからリモートへ反映。
  - *プル (Pull)*: リモートからローカルへ取得。
  - *ブランチ (Branch)*: 作業を枝分かれさせて並行開発。
  - *.gitignore*: Gitの管理から除外するファイル（`.env`や`node_modules`）を指定。

## 6. AI APIとエージェント開発
AIサービスの裏側を理解し、自作のAIアプリやエージェントを構築する技術。

### AI APIの基本
プログラムから直接AIモデルを操作する仕組み。公式SDK（例: `@google/genai`）を用いるのが推奨。
- **システムプロンプト (`systemInstruction`)**: AIの性格や役割を定義。
- **Temperature**: 出力のランダム性（創造性）を調整。0（安定）〜 2（創造的）。
- **Structured Output**: 出力をJSONなどの決まった形式に強制。
- **Tool Calling**: AIに外部機能（Web検索など）の使用を提案させ、プログラム側で実行する仕組み。

### AIエージェント
- ユーザーの代わりに**自律的**にタスクを遂行するシステム。
- **ReAct (Reason + Action)**: 「思考 → 行動 → 観察」のサイクルを繰り返してタスクを完遂するフレームワーク。
- **コンテキストエンジニアリング**: AIの各ステップで、どのような文脈情報を渡すかを設計する技術。
