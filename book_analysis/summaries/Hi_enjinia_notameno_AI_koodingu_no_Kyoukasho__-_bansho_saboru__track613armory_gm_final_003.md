## ## 2. プログラミング基礎概念
### ### 変数とデータ型
- **変数**: データを保存する「箱」
  - `const`: 再代入不可の定数（基本推奨）
  - `let`: 再代入可能な変数
  - `const`でも配列やオブジェクトの中身は変更可能（push等）

- **データ型**:
  - **文字列**（String）: `"こんにちは"` または `'太郎'`
  - **数値**（Number）: 整数と小数を区別しない
  - **真偽値**（Boolean）: `true` または `false`
  - **配列**（Array）: 複数データのまとまり（インデックスは0から開始）
  - **オブジェクト**（Object）: キーと値のペア（辞書のような構造）

- **型変換の注意点**:
  - `100 + "200"` → `"100200"`（文字列結合）
  - `Number("25") + 1` → `26`（明示的型変換）

### ### テンプレートリテラル
- バッククォート（`）で文字列を囲むことで、`${変数名}`で変数を埋め込み可能
- 改行もそのまま記述可能
  ```javascript
  const name = "太郎";
  console.log(`こんにちは、${name}さん！`);
  ```

### ### 条件分岐
- **if文**: 条件に応じて処理を分岐
  ```javascript
  if (条件式) {
    // 条件が真のときの処理
  } else if (別の条件) {
    // 別の条件が真のときの処理
  } else {
    // どの条件も満たさないときの処理
  }
  ```

- **比較演算子**: `===`（厳密等価）、`!==`、`>`、`<`、`>=`、`<=`
- **論理演算子**: `&&`（AND）、`||`（OR）、`!`（NOT）
- **配列やオブジェクトのチェック**:
  - `配列.includes(要素)`: 要素が含まれるか確認
  - `"キー" in オブジェクト`: キーの存在確認

### ### ループ（繰り返し処理）
- **for...of文**: 配列の要素を1つずつ取り出して処理
  ```javascript
  for (const item of array) {
    // 処理
  }
  ```

- **Object.entries()**: オブジェクトをループ処理するための変換
  ```javascript
  for (const [key, value] of Object.entries(obj)) {
    // 処理
  }
  ```

- **while文**: 条件が満たされている間繰り返す（無限ループに注意）

### ### 関数
- **関数定義**: `function` キーワードまたは**アロー関数**（`=>`）
  ```javascript
  // 従来の関数
  function greet(name) { return `こんにちは、${name}さん！`; }
  
  // アロー関数
  const greet = (name) => `こんにちは、${name}さん！`;
  ```

- **引数**: 関数への入力値
- **戻り値**: `return`で処理結果を返す
- **デフォルト引数**: 省略時の初期値を設定可能
  ```javascript
  function greet(name, greeting = "こんにちは") {
    return `${greeting}、${name}さん！`;
  }
  ```

### ### 組み込み機能
- **配列メソッド**:
  - `push()`: 要素の追加
  - `indexOf()`: 要素のインデックス取得
  - `splice()`: 要素の削除
  - `filter()`: 条件で絞り込み
  - `reduce()`: 累積処理（合計値など）
  - `sort()`: 並べ替え

- **文字列メソッド**:
  - `toUpperCase()`: 大文字変換
  - `toLowerCase()`: 小文字変換
  - `split()`: 文字列の分割
  - `replace()`: 文字列の置換

## ## 3. APIの基礎と実践
### ### APIとは
- **Application Programming Interface**の略
- ソフトウェア間の「公式なルールブック」
- ブラウザを開かずにChatGPTなどのサービスを直接呼び出し可能

### ### APIの利点
- **自動化の安定性**: UI変更の影響を受けにくい
- **事前通知**: 仕様変更時は告知がある
- **スケジュール実行**: 毎日決まった時間に自動実行可能

### ### APIの認証とセキュリティ
- **APIキー**: パスワードと同等以上に機密性が高い
- **環境変数管理**: `.env`ファイルにAPIキーを保存し、コードから分離
- **セキュリティの鉄則**:
  - APIキーをコードに直接書き込まない（ハードコーディングの禁止）
  - `.gitignore`に`.env`を記述し、Git管理から除外

### ### HTTPリクエストの基本
- **fetch関数**: JavaScriptでのHTTPリクエスト送信
- **リクエストの構成要素**: URL、リクエストボディ、ヘッダー（APIキー含む）
- **HTTPメソッド**: GET（取得）、POST（作成）、PUT（更新）、DELETE（削除）
- **ステータスコード**:
  - 200番台: 成功
  - 400番台: クライアント側のエラー
  - 500番台: サーバー側のエラー

### ### APIの料金体系
- **従量課金制**: トークン単位での課金
- **トークン**: AIがテキストを処理する最小単位
  - 日本語: 約1文字 = 0.5〜0.7トークン
- **節約の仕組み**:
  - **プロンプトキャッシュ**: 同じプロンプトの再利用
  - **Batch API**: 非同期処理で50%割引

### ### ガードレール（安全機構）

## ## 6. CLI（コマンドライン）の基礎
### ### CLIのメリット
1. **高速性**: コマンド一発で作業完了
2. **AIとの相性**: テキストで操作するためAIに指示を出しやすい
3. **自動化のしやすさ**: 複雑な操作もスクリプトとして記録可能

### ### 基本的な6つのコマンド
- **cd**: ディレクトリの移動
- **ls**: ファイルやディレクトリの一覧表示
- **mkdir**: ディレクトリの作成
- **mv**: ファイルやディレクトリの移動・名前変更
- **cp**: ファイルやディレクトリのコピー
- **rm**: ファイルやディレクトリの削除（**復元不可**）

### ### 作業効率化のテクニック
- **↑キー**: 過去のコマンド履歴の呼び出し
- **Tab補完**: ファイル名やフォルダ名の自動補完
- **AIコマンド生成**: 日本語で「やりたいこと」を伝えるだけで適切なコマンドを生成

### ### CLIツールの活用例
- **qpdf**: PDFの結合・分割・パスワード設定
- **ffmpeg**: 動画の圧縮・切り出し・形式変換
- **Gemini CLI**: プロジェクトのREADME.md自動生成

## ## 7. GitとGitHub
### ### Gitの基本概念
- **バージョン管理**: 変更を記録していつでも過去の状態に戻れる仕組み
- **分散型**: 高速で柔軟な操作が可能

- **コミット**: 変更内容の確定と記録
- **ステージング**: コミット前の準備段階
- **ブランチ**: 作業の枝分かれ（実験的変更を安全に試せる）
- **プッシュ**: ローカルの変更をリモート（GitHub）に反映
- **プル**: リモートの変更をローカルに取得

### ### GitHubの活用
- **クラウドバックアップ**: 別のPCからもアクセス可能
