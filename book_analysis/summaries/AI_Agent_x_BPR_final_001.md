## 2. AIエージェントのアーキテクチャと定義
### AIエージェントの4つの基本要件
AIエージェントは「目的」「理解」「行動」「学習」を自律的に行う存在として定義され、現在は人間が設定したゴールの範囲内で最適化する「半自律型」が主流である。

### 5層構造の技術スタック
AIエージェントは以下の5つの層（レイヤー）の統合によって構成される。
1. **知能層**：`LLM`（言語理解・推論の中核）
2. **記憶・知識層**：コンテキスト（短期記憶）、パラメータ（長期記憶）、`RAG`（外部データ参照）
3. **行動層**：API連携やツール呼び出しによる実際のタスク実行
4. **制御・調整層**：タスクの統率、ガードレールの設定
5. **学習・最適化層**：フィードバックによる継続的な改善サイクル

### 導入形態（アーキテクチャ）の分類
- **SaaS型**：初期導入が容易だがカスタマイズ性とセキュリティに制限がある。
- **API連携型**：既存システムと容易に連携可能。UIやロジックを自社設計できる。
- **ローカルLLM型**：オープンソースモデルを自社サーバーで構築。機密情報を扱う際のセキュリティが最も高いが、運用コストが重い。

### AIエージェント特有の開発アプローチ
従来のシステム開発（脱積み上げ・脱正確さ・脱サイロ化）とは異なり、確率的な出力に対する「ヒューマン・イン・ザ・ループ」の設計や、業務部門と開発部門の密な並走（協働）が求められる。

## 3. 業務改革の推進手法：AI AM法
### トップダウンとボトムアップの両輪
失敗するDXの多くは「現場任せ」や「目的喪失」に陥っている。改革を成功させるには以下の構造が必要である。
- **経営層**：具体的な成果目標（KGI）と「撤退基準」を明示する。
- **変革リーダー**：経営目標を現場の行動指標に翻訳し、横串を通す。
- **現場層**：ユースケースを具現化し、小さな成功（クイックウィン）を積み上げる。

### AI AM（AI Agent Management）法：5つのフェーズ
1. **基本計画立案**：課題抽出、KPI設定、ステークホルダーの合意形成。
2. **業務設計**：`As-Is`（現状）分析から`To-Be`（あるべき姿）の設計。AI前提のプロセス再構築。
3. **検証計画策定**：技術的評価と業務的評価のゴール設定。
4. **構築・検証**：小さく速く回す改善サイクル。
5. **システム開発**：本番運用（インフラ化）に向けた非機能要件の実装と役割分担の確定。

## 4. プロジェクト実践における「壁」と解決策
### 情報の壁（ガバナンスとセキュリティ）
- AIに読み込ませてはいけない情報（**個人情報**と**営業秘密**）の境界線を定義し、「許容・要承認・禁止」の3つに分類するガバナンス設計が不可欠。シャドーIT（シャドーAI）による情報漏洩リスクを防ぐ必要がある。

### 精度の壁（LLMの確率的性質）
- `LLM`は次に来る単語を確率的に予測する仕組みであり、必ずハルシネーション（もっともらしい嘘）が発生する。
- **「品質100%を狙わない」**：確率的な振る舞いを受け入れ、`Human-in-the-Loop`（人の介在）により「そこそこ使えるレベル」を担保する業務設計を行う。
- **プロンプトエンジニアリングの要点**：思考の連鎖、Few-shotプロンプト、表データの前処理などにより、AIの「アテンション（目線）」を適切に誘導する。

### 評価の壁（検証地獄とROIの可視化）
- **検証地獄**から抜け出すためには、技術評価と業務評価を明確に分離し、「何をもって実運用に移すか」の定量基準を設ける。
- **ROIの算出**：
  - `工数削減費 = 外部委託削減費 + 社内人件費削減費`
  - 個別のROIがマイナスでも、**横展開（スケール）による波及効果**を含めて投資判断を行う。

### 人と組織の壁（消極的抵抗とIT部門との連携）
- 最大の障壁は「消極的な抵抗勢力」。改革を「自分ごと」にさせるためのクイックウィンの提示や、社内インフルエンサーを活用した口コミ戦略が有効。
- **IT部門の巻き込み**：初期段階からIT部門を参加させ、開発・運用・保守の責任分解点を明確にする。

## 5. ユースケース選定と適用領域
### 業務アセスメント（5つのステップ）
適用すべき業務を科学的に選定する。
1. **As-Is業務工数の整理**：処理頻度と所要時間の把握。
2. **AI活用適性の評価**：要約・抽出、文章生成などAIが得意なタスクを含むか（0/1評価）。
3. **AI活用前提条件の確認**：精度要求レベル、業務標準化度など、導入の土台が整っているか。
4. **削減率算定**：カテゴリ別の削減率を用いた効果の定量見積もり。
5. **取り組み業務の決定**：効果と拡張性のバランスによる優先順位付け。

### バックオフィス業務（最優先適用領域）
- **要約・転記型**：非構造化データの構造化・転記（例：議事録作成、履歴書入力）。
- **チェック・校閲型**：ルールベースの突合・審査・一次チェック（例：経費精算、請求書処理）。
- **問い合わせ対応型**：`RAG`を活用した社内FAQボットによる一次対応。

### フロントオフィス業務
- **カスタマーサポート**：一次対応の自動化、オペレーション処理の代行。
- **セールス・マーケティング**：リード獲得の自動化、提案書ドラフト作成、キャンペーン分析・最適化。
- **人の判断との分担**：「手順が明確で判断の重要性が低い業務」からAI化を進め、「手順が曖昧で重要性が高い業務」は人が担う。
