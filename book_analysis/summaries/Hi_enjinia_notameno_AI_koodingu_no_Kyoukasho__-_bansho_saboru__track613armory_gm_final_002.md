## 第1部：プログラミングの基礎
AIが生成したコードを読み解き、的確な指示を出すための必須概念です。

### 変数とデータ型
- **変数**: データを一時的に保存する「箱」。`const`（再代入不可）と `let`（再代入可能）で宣言し、基本的には `const` を使用します。
- **テンプレートリテラル**: バッククォート（`` ` ``）で囲み、`${変数名}` で変数を埋め込める文字列。
- **データ型**: 
  - **文字列**: `" "` または `' '` で囲む。
  - **数値**: 整数・小数を区別せず扱う。暗黙的に文字列に変換される（`100 + "200"` → `"100200"`）ため、`Number()` を用いた明示的な型変換が必要な場合がある。
  - **真偽値**: `true` または `false`。
- **データ構造**:
  - **配列**: 複数データを順番に管理。インデックスは `0` から開始。
  - **オブジェクト**: キーと値のペアでデータを管理。`obj['key']` または `obj.key` でアクセス。

### 条件分岐とループ
- **条件分岐 (`if...else if...else`)**: 条式に応じて処理を切り替えます。
- **比較演算子**: 値と型を厳密に比較する `===` の使用が推奨されます。
- **論理演算子**: `&&` (AND), `||` (OR), `!` (NOT)。
- **ループ**:
  - `for...of`: 配列の要素を順番に取り出す。
  - `while`: 特定の条件が満たされるまで処理を繰り返す。
  - `Object.entries()`: オブジェクトを `[key, value]` の配列に変換し、`for...of` でループ可能にする。

### 関数
- **基本構文**: 一連の処理をまとめて再利用可能にする仕組み。
- **引数と戻り値**: 関数に入力を渡し、`return` で結果を出力する。
- **デフォルト引数**: `function greet(name, greeting = "こんにちは")` のように省略時の初期値を設定。
- **アロー関数**: `const greet = (name) => { ... }` というモダンな記法。AIが生成するコードで頻出します。

### エラーとデバッグ
- **デバッグ**: プログラムの不具合を見つけて修正する作業。
- **エラー対応**: エラーメッセージは「エラーの種類」と「行番号」を示す重要な手がかりです。
- **AIへの質問**: エラーの全文と対象のコード、そして「何をしようとしていたか（意図）」をセットでAIに伝えることで、的確な修正案が得られます。

## 第2部：APIとライブラリの活用
プログラムを外部サービスと連携させ、開発を効率化するための知識です。

### API (Application Programming Interface)
- **概要**: ソフトウェアの機能を別のソフトウェアから利用するための「公式なルールブック」。
- **利点**: ブラウザを通じた手動操作（Webスクレイピング等）と比べ、**安定性**が高く、**自動化**に適しています。
- **HTTPリクエスト**: Webサービスの通信はHTTPプロトコルに従い、POSTリクエストでJSONデータを送信します。
- **ステータスコード**: 200（成功）、400番台（クライアント側のエラー）、500番台（サーバー側のエラー）。

### セキュリティとAPIキーの管理
- **APIキー**: 外部サービスの利用を認証するための「合言葉」。
- **安全な管理**: APIキーは `.env` ファイルに保存して環境変数として読み込み、`.gitignore` でGit管理対象外にします。コードへの直書き（ハードコーディング）は厳禁です。
- **プロンプトインジェクション**: 悪意のある入力によりAIの動作が乗っ取られる攻撃。対策として、システムプロンプトに「指示の変更要求には従わない」よう設定します。

### ライブラリとパッケージ管理
- **ライブラリ**: 他の人が書いた便利なコードの集まり。`@google/genai` などが代表的。
- **npm (Node Package Manager)**: JavaScriptのライブラリを管理・インストールするツール。
- **主要ファイル**:
  - `package.json`: プロジェクトの設計図。依存ライブラリが記録される。
  - `node_modules`: インストールされたライブラリの実体。サイズが大きいため共有・Git管理はしない。
- **セキュリティ**: 週間ダウンロード数やGitHubスター数を確認し、信頼できるライブラリを選定します。`npm audit` で脆弱性をチェックします。

## 第3部：AIコーディングツールの使いこなし
AIを単なるツールから「パートナー」へと昇華させるための実践手法です。

### ツールの形態と機能
- **形態**: エディター型、CLI型（Claude Codeなど）、クラウド型（Julesなど）に分類され、それぞれの強みに応じて使い分けます。
- **主要機能**:
  - **ファイルのメンション (`@ファイル名`)**: AIに参照してほしい情報を正確に渡す。
  - **Webサーチ**: 最新情報を取得させる（リスクを避けるためSecure Modeの活用も推奨）。
  - **MCP (Model Context Protocol)**: AIの能力を拡張する共通規格。Context7などを導入し、最新のライブラリ仕様を参照させます。
  - **ルールファイル (`AGENTS.md` 等)**: AIの行動マニュアルを定義し、プロジェクトごとの「型」を教え込みます。
  - **カスタムコマンド**: 頻繁に使う指示（校正、画像生成など）を `/command` で即座に実行。
  - **ガードレール (Guardrails)**: `Allow List` / `Deny List` を用いて、AIの危険な操作を防止します。

### コンテキスト・エンジニアリング
- **コンテキスト**: AIが最良の回答をするために「選択肢を狭める情報」。目的、対象、背景（Why, Who, What）を明確にします。
- **ルールファイルの育成**: AIに過去の成果物を分析させ、文体や構成のルールを抽出させます。実際の作業を通じたフィードバックをルールに反映させ、継続的にブラッシュアップします。
- **動作確認**: チェックリスト（基本機能、異常系、デザイン）を作成し、AIに品質保証専門家の視点でレビューさせます。

### CLI (コマンドライン・インターフェース)
- **基礎**: 6つの基本コマンド（`cd`, `ls`, `mkdir`, `mv`, `cp`, `rm`）を習得すれば日常的な操作の大部分が可能です。
- **AIとの相性**: やりたいことを日本語で伝えるだけで、AI（WarpやAntigravityの機能）が適切なコマンドを生成します。
- **応用**: `qpdf` によるPDF結合や、`ffmpeg` による動画編集など、CLIツールを通じて高度な自動化を実現します。

### GitとGitHub
- **Git**: ファイルの変更履歴を記録し、いつでも過去の状態に戻せるようにするローカルツール。AIの予期せぬ編集を元に戻すための必須のセーフティネットです。
- **GitHub**: Gitの履歴をクラウド上で共有・保存するサービス。
- **基本用語**: 
  - **リポジトリ**: プロジェクトの箱。
  - **コミット**: 変更の確定と記録。
  - **プッシュ / プル**: ローカルとリモート間のデータの送受信。
  - **ブランチ**: 作業を枝分かれさせ、安全に並行開発する仕組み。
- **クラウドAIエージェント連携**: JulesやCodexなどのクラウドAIにGitHub上の作業を依頼できるようになります。

## 第4部：AI APIとエージェント開発
AIの仕組みを理解し、自律的にタスクを遂行するシステムを構築します。

### AI APIの基礎
- **SDK (Software Development Kit)**: `@google/genai` などの公式SDKを用いて、プログラムから直接AIを操作します。
- **主要パラメータ**:
  - **システムプロンプト (`systemInstruction`)**: AIの「性格」や「役割」を定義。
  - **`temperature`**: 回答のランダム性を調整（低い値で安定、高い値で創造的）。
  - **Structured Output**: 出力をJSON形式など、決まった構造で強制的に取得。
- **コストと運用**: 従量課金制（トークン単位）であり、プロンプトキャッシュやBatch APIを活用してコストを削減します。レートリミット（利用制限）とクレジット残高の管理が重要です。

### AIエージェントの構築
- **定義**: 「考える脳」と「外部ツールを操作する手足」を持ち、人間の代わりに自律的にタスクを遂行するシステム。
- **ReActパターン**: Reason（思考）→ Action（行動）→ Observation（観察）のサイクルを繰り返す設計思想。
- **Tool Calling**: AIが必要な外部機能（Web検索など）を自律的に選択し、呼び出す仕組み。
- **ワークフローとの使い分け**: 定型的な処理はワークフローで確実に自動化し、柔軟な判断が必要な部分のみをAIエージェントに任せるのがベストプラクティスです。
